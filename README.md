# Hello Slime CPP

VSC 환경에서 C++의 절차적, 객체지향적 흐름을 이해하기 위해 제작한 첫 번째 토이 프로젝트임.

## 🛠️ 기술 스택 및 환경 (Tech Stack & Environment)

- **Language**: C++
- **IDE**: Visual Studio Code
- **Compiler**: GCC (`g++`)
- **Version Control**: Git, GitHub

## 🧠 핵심 구현 기술 (Key Technologies)

- **Object-Oriented Programming (OOP)**: 클래스 상속(Inheritance), 가상 함수(Virtual Function)를 활용한 다형성(Polymorphism), 추상화 구현.
- **Design Patterns**: 몬스터 생성을 전담하는 팩토리 패턴(Factory Pattern) 및 직업별 전투 방식을 분리하는 전략 패턴(Strategy Pattern) 적용.
- **Data Structures (STL)**:
  - `std::vector`: 인벤토리 시스템의 동적 아이템 배열 관리.
  - `std::map`: 멤버 포인터를 활용한 세이브 데이터 메타 레지스트리 구축.
- **File I/O**: `std::fstream`을 활용한 게임 스탯 및 동적 인벤토리 영구 저장/불러오기(Save/Load) 로직 구현.
- **Data-Driven Design**: 매직 스트링(Magic String)을 배제하고 아이템/몬스터의 고유 `Type` 코드를 기반으로 작동하는 시스템 설계.

- **데이터 무결성 검증(Checksum) 시스템 적용**: 유저의 로컬 세이브 파일(`savefile.txt`) 임의 조작을 방지하기 위해 핵심 스탯(레벨, 골드, 체력)을 기반으로 고유 해시값을 산출하는 체크섬 암호화 로직을 구현함. 로드 시 데이터 변조가 적발될 경우 재화를 몰수하는 안티 치트(Anti-Cheat) 페널티 연동.
- **업적(Achievement) 시스템 연동**: 플레이어의 누적 몬스터 처치 수와 소지 재화를 실시간으로 추적하는 이벤트 리스너 형태의 `checkAchievements()` 함수를 구현함. 특정 조건 도달 시 영구적인 패시브 스탯(STR, VIT)을 부여하여 유저의 장기 리텐션을 유도하는 실무적 동기부여 로직 추가.
- **아이템 희귀도(Rarity) 및 전설 아이템 추가**: 보스 몬스터 처치 시 낮은 확률(5%)로 최상위 등급인 '전설의 장비'가 드랍되는 로직을 구현하여, 파밍의 재미와 엔드 콘텐츠 보상을 강화함.
- **안전 종료(Safe Exit) 로직 구현**: 마을에서 게임 종료(9번) 선택 시 자동으로 `player.save()`를 호출하도록 수정하여, 유저의 조작 실수로 인한 데이터 증발을 원천 차단하는 실무형 방어 로직 적용.
- **경제 밸런싱 및 스탯 초기화(Stat Reset) 시스템 추가**: 잉여 골드의 소비처(Gold Sink) 역할을 수행하는 500G 비용의 스탯 롤백 기능을 구현함. 투자된 스탯으로 증가했던 최대 체력 및 마나의 동적 역산 로직을 적용하여 데이터 무결성을 보장함.
- **전술적 도주 페널티(Flee Penalty) 도입**: 전투 중 도망가기 선택 시 보유 골드의 10%를 상실하는 페널티 로직을 구현하여, 무분별한 전투 회피를 방지하고 리스크 관리의 재미를 더함.
- **최대 레벨(Max Level) 하드캡 적용**: `Player` 클래스의 경험치 획득 로직에 레벨 50 제한을 두어, 무한 레벨업으로 인한 오버플로우 버그를 방지하고 실무적인 밸런스 캡(Cap)을 구현함.
- **전투 난수 변수(RNG Variance) 시스템 도입**: `Battle` 클래스의 교전 페이즈에 10%의 공격 회피율과 15%의 치명타(1.5배 피해) 확률 공식을 추가하여, 확정 데미지로 인한 전투의 단조로움을 탈피하고 실무 수준의 전투 설계 디테일을 확보함.
- **전투 시스템 고도화 및 엔딩 구현**: `Battle` 시스템에 턴 기반 상태 이상(독 피해, 기절) 로직을 적용하여 전투의 전략성을 강화함. 100층 도달 시 하드코딩된 오버 스탯의 최종 보스 '마왕'이 등장하며, 처치 시 프로세스가 종료되는 텍스트 기반 엔딩 크레딧 시퀀스를 구현함.
- **오토 세이브(Auto-save) 시스템**: 전투 승리 직후 `player.save()`를 강제 호출하도록 로직을 수정하여, 예기치 않은 종료 시에도 플레이어의 진행 데이터와 인벤토리가 안전하게 보존되도록 구성함.
